# Plan 23-01: Ingredient Schema Migration

## Objective

Create DB migration 026 that adds the normalized ingredient catalog tables and columns, plus TypeScript types for all new/extended entities.

## Context

- Baseline: migration 025 applied (order_items.notes, tenant_settings.item_notes_enabled, Realtime publication)
- Pattern: All migrations use IF NOT EXISTS guards and are applied via Supabase SQL Editor
- RLS pattern: tenant-scoped policies use auth_tenant_id() helper (see migration 019)
- Types file: src/types/database.ts — extend in place

## Tasks

### Task 1: Create migration 026 — ingredient tables and columns
**type: auto**

Create `supabase/migrations/026_ingredient_schema.sql` with:

1. `ingredients` table:
   - id UUID PK default gen_random_uuid()
   - tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE
   - name TEXT NOT NULL
   - image_url TEXT
   - default_extra_price NUMERIC(10,2) DEFAULT 0
   - default_add_price NUMERIC(10,2) DEFAULT 0
   - is_available BOOLEAN DEFAULT true
   - position INTEGER DEFAULT 0
   - translations JSONB DEFAULT '{}'
   - created_at TIMESTAMPTZ DEFAULT now()
   - updated_at TIMESTAMPTZ DEFAULT now()
   - RLS enabled; policy "Tenant members manage ingredients" (all ops, tenant-scoped); policy "Public read available ingredients" (SELECT, is_available=true)
   - Index: idx_ingredients_tenant ON ingredients(tenant_id)
   - updated_at trigger (same pattern as tenant_settings)

2. `product_ingredients` join table:
   - product_id UUID NOT NULL REFERENCES products(id) ON DELETE CASCADE
   - ingredient_id UUID NOT NULL REFERENCES ingredients(id) ON DELETE CASCADE
   - tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE
   - is_default BOOLEAN DEFAULT false
   - extra_price_override NUMERIC(10,2)
   - add_price_override NUMERIC(10,2)
   - position INTEGER DEFAULT 0
   - PRIMARY KEY (product_id, ingredient_id)
   - UNIQUE(product_id, ingredient_id)
   - RLS enabled; policy "Tenant members manage product_ingredients" (all ops, tenant-scoped); policy "Public read product_ingredients" (SELECT)
   - Index: idx_product_ingredients_product_tenant ON product_ingredients(product_id, tenant_id)

3. `tenant_settings.ingredient_customization_enabled` column:
   - BOOLEAN NOT NULL DEFAULT false
   - ADD COLUMN IF NOT EXISTS

4. `order_items.ingredient_modifications` column:
   - JSONB (nullable, no default)
   - ADD COLUMN IF NOT EXISTS

All wrapped in DO $$ BEGIN ... IF NOT EXISTS ... END $$ for idempotency on policies.

**Done criteria:** File exists at supabase/migrations/026_ingredient_schema.sql with all 4 schema changes

### Task 2: Add TypeScript types for ingredient entities
**type: auto**

Extend `src/types/database.ts` with:

1. `Ingredient` interface:
   ```ts
   export interface Ingredient {
     id: string
     tenant_id: string
     name: string
     image_url: string | null
     default_extra_price: number
     default_add_price: number
     is_available: boolean
     position: number
     translations: Record<string, { name?: string }>
     created_at: string
     updated_at: string
   }
   ```

2. `ProductIngredient` interface:
   ```ts
   export interface ProductIngredient {
     product_id: string
     ingredient_id: string
     tenant_id: string
     is_default: boolean
     extra_price_override: number | null
     add_price_override: number | null
     position: number
   }
   ```

3. `IngredientModification` types (for JSONB structure):
   ```ts
   export interface IngredientRemoval {
     ingredient_id: string
     name: string
   }
   export interface IngredientExtra {
     ingredient_id: string
     name: string
     qty: number
     unit_price: number
   }
   export interface IngredientModifications {
     removed: IngredientRemoval[]
     extras: IngredientExtra[]
     added: IngredientExtra[]
   }
   ```

4. Extend `TenantSettings` interface: add `ingredient_customization_enabled: boolean`

5. Extend `OrderItem` interface: add `ingredient_modifications: IngredientModifications | null`

6. Add joined type:
   ```ts
   export interface ProductIngredientWithIngredient extends ProductIngredient {
     ingredient: Ingredient
   }
   ```

**Done criteria:** All interfaces present in database.ts, TenantSettings and OrderItem extended correctly
