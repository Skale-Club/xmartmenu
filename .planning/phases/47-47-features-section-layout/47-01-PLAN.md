# Phase 47 — Features Section Layout

## Context

The features section in `ClientPage.tsx` currently renders a 2-column grid at all non-phone breakpoints, uses `ShoppingCart` for the Online Ordering card icon, and has an oversized subtitle. Phase 45 (complete) already defined `FoodDrinkCombo` in the same file and registered it in `getIcon()`. This phase makes four targeted edits to that file only.

**Exact targets identified in `src/app/(marketing)/ClientPage.tsx`:**
- Line 369: `<div className="grid grid-cols-1 md:grid-cols-2 gap-6">` — FEAT-01 + FEAT-02 grid
- Line 378: `className="bg-white/5 backdrop-blur-sm border border-white/10 rounded-[1.25rem] p-8 relative overflow-hidden group"` — FEAT-02 card padding
- Line 387: `<h3 className="text-2xl font-bold text-white mb-3">` — FEAT-02 card title
- Line 83: `icon: ShoppingCart,` in the hardcoded `features` array — FEAT-03
- Line 5: `ShoppingCart` in the lucide-react import — FEAT-03 cleanup (only if ShoppingCart has no other uses)
- Line 366: `<p className="text-xl text-zinc-400">{sectionSubtitle}</p>` inside `FeatureBlocks` — FEAT-04

Note: `ShoppingCart` also appears at line 133 inside `getIcon()` iconMap. It must remain there. Only the `features` array reference at line 83 is swapped; the import stays.
