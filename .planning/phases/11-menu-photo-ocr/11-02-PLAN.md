# Plan 11-02: OCR Menu Route + UI Integration

## Objective

Implement the POST `/api/superadmin/tenants/[id]/ocr-menu` route that receives a `storagePath` (from the signed upload completed in Plan 01), downloads the image from Supabase Storage, calls GPT-4.1-mini vision via `generateObject`, writes extracted categories and products directly to the DB, logs `ai_usage`, and invalidates the ISR cache. Also adds the OCR photo upload UI block to `TenantDetailClient`.

## Context

- `@.planning/phases/11-menu-photo-ocr/11-01-SUMMARY.md` — Infrastructure: @ai-sdk/openai installed, OcrMenuSchema + OcrMenuResult in schemas.ts, GET ocr-upload-token route
- `@src/app/api/superadmin/tenants/[id]/seed/route.ts` — Pattern reference for DB writes, ai_usage logging, revalidatePath
- `@src/app/(superadmin)/tenants/[id]/TenantDetailClient.tsx` — UI to extend with OCR photo upload section

## Tasks

### Task 1 — Create POST ocr-menu route (AI-11, AI-12)
**type: auto**

Create `src/app/api/superadmin/tenants/[id]/ocr-menu/route.ts`:

- `export const runtime = 'nodejs'` — Node.js required for @ai-sdk/openai
- `export const maxDuration = 60` — vision calls typically 5–15s
- Auth: `assertSuperadmin()` guard
- Body: `{ storagePath: string, menuId: string }`
- Download image from Supabase Storage using `service.storage.from('tenant-assets').download(storagePath)` → get `Blob`
- Convert `Blob` to base64 data URL for AI SDK vision message
- Call `generateObject({ model: openai('gpt-4.1-mini'), schema: OcrMenuSchema, messages: [...] })` with image content parts
- Write categories + products to DB (additive-only, skip existing names — same pattern as seed route)
- `price ?? 0` for all products — AI-12: failed parsing saved as 0
- Log `ai_usage` with `feature_key: 'ocr_menu'`
- `revalidatePath` for tenant/menu slugs
- Return `{ success: true, categoriesCreated, productsCreated, tokensUsed }`

### Task 2 — Add OCR upload UI to TenantDetailClient (AI-10, AI-11)
**type: auto**

Extend `src/app/(superadmin)/tenants/[id]/TenantDetailClient.tsx` with an OCR subsection inside the AI Tools block:

- New state: `ocrFile`, `ocrLoading`, `ocrStatus`
- File input (`accept="image/*"`) for selecting menu photo
- "Upload & Extract" button: 
  1. GET `ocr-upload-token?filename={file.name}` → `{ uploadUrl, storagePath }`
  2. PUT file directly to `uploadUrl` (Storage signed URL)
  3. POST `ocr-menu` with `{ storagePath, menuId: selectedMenuId }`
  4. Show result message with counts or error
- Loading state while processing ("Extracting menu — this may take up to 30 seconds...")
- Success/error banner consistent with existing seed status banners
