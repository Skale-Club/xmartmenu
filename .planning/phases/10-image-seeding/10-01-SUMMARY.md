### Task 2: seed-image route created

Created `src/app/api/superadmin/tenants/[id]/seed-image/route.ts` with:

- `export const runtime = 'nodejs'` — Sharp native bindings, no Edge
- `export const maxDuration = 300` — separate file from seed/route.ts (which uses 60); file-scoped constraint in Next.js
- Three seed types in a single POST handler via `type` discriminator:

| Type | Description | Aspect Ratio | DB Target |
|------|-------------|--------------|-----------|
| `image_cover` | One banner per tenant, additive (skip if banner_url set) | `4:1` | `tenant_settings.banner_url` |
| `image_products` | Bulk: all products where image_url IS NULL, position order | `1:1` | `products.image_url` |
| `image_single_product` | One product by productId, additive (skip if image_url set) | `1:1` | `products.image_url` |

All types: generate via `generateImage({ model: google.image('gemini-3.1-flash-image-preview') })` → `convertBufferToWebP()` → Supabase Storage `tenant-assets` (upsert: true, contentType: image/webp) → DB write → `ai_usage` upsert → `revalidatePath()`.

Commit: `ebf81db`

## Decisions Made

1. **Separate route for image seeding** — `maxDuration` is file-scoped in Next.js; image generation needs 300s, text seed uses 60s. Folding into `/seed` route was not possible.
2. **`4:1` cover aspect ratio** — BrandingClient recommends 3:1 display ratio, but Nano Banana 2 does not support `3:1` as an `aspectRatio` value. `4:1` is the closest wide supported ratio.
3. **`1:1` product aspect ratio** — Safe default for product cards that use `object-cover`; avoids layout shifts.
4. **Write to `products.image_url` (singular)** — Migration 019 defines both `image_url` (text, canonical) and `image_urls` (text[], legacy). Public menu renders `image_url`. Per-product AI seeding writes to `image_url` only.
5. **ai_usage logged inside per-product loop** — Each Nano Banana 2 call is a separate billable event; upsert accumulates `call_count` and `token_count` per `(tenant_id, feature_key, date)`.
