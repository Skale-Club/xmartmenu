    ```typescript
      plan: Pick<EffectivePlan, 'monthly_price' | 'annual_price' | 'billing_cycle' | 'status'>

      eventId?: string                                     // Stripe event.id; absent for onboarding
      tags?: string[]                                      // status/direction tags from the producer
    }
    ```

    Implement and export three functions:

    1. `normalizeMrr(plan)` — `plan.billing_cycle === 'annual' ? plan.annual_price / 12 : plan.monthly_price`. This is the ONLY MRR computation; never read raw `plans.monthly_price`.

    2. `selectStage(reason, status): XphereStage` using `XPHERE_STAGES` (reason takes priority, then status):
       - `onboarded` -> `XPHERE_STAGES.ONBOARDING`
       - `plan_activated` -> `XPHERE_STAGES.ACTIVE`
       - `past_due` -> `XPHERE_STAGES.AT_RISK`
       - `churned` -> `XPHERE_STAGES.CHURNED`
       - otherwise fall back on subscription `status`: `active` -> ACTIVE, `past_due` -> AT_RISK, `cancelled` -> CHURNED, `trial` -> ONBOARDING.

    3. `buildSyncPayload(input: SyncMapInput): XphereSyncRequest` assembling:
