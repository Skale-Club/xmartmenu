**Step B — If size is ABOVE 300000 bytes, optimize the image.**

The current implementation (Phase 12 decision: "flat dark CSS ImageResponse with no fetch()") should be well under 100 KB based on the flat colors used. However, if measurement shows it is over limit:

Option 1 — Reduce complexity to guaranteed-safe minimal version:
```typescript
import { ImageResponse } from 'next/og'

export const alt = 'XmartMenu — Your restaurant menu, online in minutes'
export const size = { width: 1200, height: 630 }
export const contentType = 'image/png'

export default function Image() {
  return new ImageResponse(
    (
      <div
        style={{
          background: '#18181b',
          width: '100%',
          height: '100%',
          display: 'flex',
          flexDirection: 'column',
          alignItems: 'center',
          justifyContent: 'center',
          fontFamily: 'sans-serif',
        }}
      >
        <div style={{ fontSize: 80, fontWeight: 700, color: '#ffffff' }}>
          XmartMenu
        </div>
        <div style={{ fontSize: 30, color: '#a1a1aa', marginTop: 20 }}>
          Your restaurant menu, online in minutes
        </div>
      </div>
    ),
    { ...size }
  )
}
```

This stripped version uses only text on a flat background — guaranteed under 100 KB.

- `Disallow: /api/`
- `Disallow: /admin` (or equivalent admin paths)

- `Sitemap: https://xmartmenu.skale.club/sitemap.xml`

**3. Verify JSON-LD (SEO-03):**

`https://xmartmenu.skale.club`

```bash
curl -s https://xmartmenu.skale.club/demo | grep "ld+json"
```
