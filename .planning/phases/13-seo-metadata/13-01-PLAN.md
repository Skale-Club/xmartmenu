**Step B — Create src/app/sitemap.ts:**

```typescript
import type { MetadataRoute } from 'next'

const BASE_URL = 'https://xmartmenu.skale.club'

export default function sitemap(): MetadataRoute.Sitemap {
  return [
    {
      url: BASE_URL,
      lastModified: new Date(),
      changeFrequency: 'monthly',
      priority: 1,
    },
  ]
}
```

CRITICAL constraints:
- List ONLY `/` — NEVER query the tenants table (would leak the full tenant roster as public XML per Pitfall 8 in PITFALLS.md)
- Do NOT call Supabase or any runtime API — this must remain a static route handler cached at build
- Do NOT add `export const dynamic = 'force-dynamic'` — the default static caching is correct
- The `lastModified: new Date()` is evaluated at build time for static routes — this is correct

**Step C — Create src/app/robots.ts:**

The ROADMAP specifies disallowing `/api/`, `/admin/`, `/superadmin/`. Map these to the actual route structure:
- `/api/` — all API routes
- `/admin/` — maps to `/(admin)/` route group (URL segment is `/admin` in the rendered path? Verify: the route group `(admin)` uses path prefix `/dashboard`, `/menu`, `/settings`, `/overview` — NOT `/admin`. Check actual admin URL structure)
- `/superadmin/` — maps to `/(superadmin)/` route group

Read the actual route group folder names before writing disallow rules. The route group `(admin)` has parentheses so it adds NO URL segment. Look at what URLs the admin routes actually serve. Based on the codebase (from research ARCHITECTURE.md), admin routes are at `/dashboard`, `/menu`, `/settings`, `/overview`, `/users`. Superadmin routes are at `/tenants`, `/users`, `/overview`.

```typescript
import type { MetadataRoute } from 'next'

export default function robots(): MetadataRoute.Robots {
  return {
    rules: [
      {
        userAgent: '*',
        allow: '/',
        disallow: [
          '/api/',
          '/dashboard',
          '/settings',
          '/menu',
          '/menus',
          '/tenants',
          '/overview',
          '/users',
          '/onboarding',
          '/auth',
          '/admin',
          '/superadmin',
        ],
      },
    ],
    sitemap: 'https://xmartmenu.skale.club/sitemap.xml',
  }
}
```

Note: `/admin` and `/superadmin` are included to satisfy the ROADMAP requirement text (SEO-02: "disallows /api/, /admin/, /superadmin/") even though the actual route group folders use `(admin)` and `(superadmin)` syntax. The URL segments `/admin` and `/superadmin` have no real routes but Disallow is harmless and satisfies the spec.

**Add inside the component function body, before the return:**
```typescript
const organization: WithContext<Organization> = {
  '@context': 'https://schema.org',
  '@type': 'Organization',
  name: 'XmartMenu',
  url: 'https://xmartmenu.skale.club',
  description: 'Digital menu platform for restaurants via QR code',
  sameAs: [],
}

const software: WithContext<SoftwareApplication> = {
  '@context': 'https://schema.org',
  '@type': 'SoftwareApplication',
  name: 'XmartMenu',
  applicationCategory: 'BusinessApplication',
  operatingSystem: 'Web',
  offers: {
    '@type': 'Offer',
    price: '0',
    priceCurrency: 'BRL',
    description: 'Grátis durante o beta',
  },
}
```

**Wrap the existing return in a fragment and prepend the scripts:**
```tsx
return (
  <>
    {/* JSON-LD structured data — per SEO-03. Inline dangerouslySetInnerHTML only.
        NEVER use next/script for JSON-LD (causes RSC hydration duplicates in React 19).
        NEVER move this to layout.tsx (would appear on all tenant pages). */}
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{
        __html: JSON.stringify(organization).replace(/</g, '\\u003c'),
      }}
    />
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{
        __html: JSON.stringify(software).replace(/</g, '\\u003c'),
      }}
    />
    {/* existing page content below — do not change anything else */}
    ...existing JSX...
  </>
)
```

ANTI-PATTERNS — these MUST NOT appear in the file after editing:
- `import Script from 'next/script'` — wrong for JSON-LD
- `<Script type="application/ld+json">` — causes RSC hydration duplicates in React 19
- JSON-LD object definitions inside `layout.tsx` — would appear on all routes including tenant menus

The `.replace(/</g, '\\u003c')` call is mandatory per the official Next.js JSON-LD guide — prevents XSS if any string field ever contains HTML.
