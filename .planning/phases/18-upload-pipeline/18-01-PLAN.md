<objective>
Enforce WebP-only uploads across every path that writes images to Supabase Storage.

Purpose: Two upload paths (admin product images, superadmin tenant assets) currently send raw PNG/JPEG bytes to the bucket. Fixing them ensures all incoming images are converted to WebP at q85 before storage, reducing storage costs and guaranteeing consistent image format. The next.config.ts change enables Next.js image optimization to serve WebP to browsers that support it.

Output:
- ProductsClient.tsx — handleImageUpload converts each File to WebP buffer before upload, uses .webp filename extension
- upload/route.ts — POST handler converts File to WebP buffer before upload, forces .webp extension
- next.config.ts — images config gains formats and deviceSizes
</objective>
