## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding |
| sm | 8px | Toggle row internal padding, icon-to-label gap |
| md | 16px | Default field spacing, inter-element gaps |
| lg | 24px | Section internal vertical rhythm |
| xl | 32px | Layout column gap |
| 2xl | 40px | Section card padding (matches existing `p-10` = 40px) |
| 3xl | 64px | Page-level `space-y-10` outer rhythm |

Exceptions:
- Toggle row uses `p-5` (20px) horizontal padding — matches existing Operations toggle rows exactly.
- Conditional field row uses `mt-4` (16px) top margin within the toggle group.
- Section card border-radius: `rounded-[1.25rem]` (20px) — project-specific, not a standard Tailwind token.

## Color

| Role | Value | Usage |
|------|-------|-------|

1. Toggle pill background when `checked` state is true (`bg-primary`)

NOT used for: toggle row hover states, section borders, field backgrounds, descriptive text.

### Toggle Row (reuse existing Operations pattern exactly)

```
div.flex.items-center.justify-between.group.p-5.rounded-[1rem].bg-zinc-50.hover:bg-zinc-100.transition-all.cursor-pointer
  div.max-w-[70%]
    p: text-sm font-black text-zinc-950 uppercase tracking-tight  ← toggle label

  button[type=button]: h-7 w-12 rounded-full transition-colors
    → active:   bg-primary
    → inactive: bg-zinc-200
    span: h-6 w-6 rounded-full bg-white shadow-lg transform transition-transform duration-300
    → active:   translate-x-5
    → inactive: translate-x-1
```

Three toggle rows in order:
1. **Dine-in** — label: "Dine-In" — description: "Customers can order for table service at your restaurant."
2. **Pick-up** — label: "Pick-Up" — description: "Customers can place orders for pick-up at your location."
3. **Delivery** — label: "Delivery" — description: "Customers can request home delivery of their orders."

### Conditional Numeric Field (reuse KDS threshold pattern exactly)

Shown inline below its parent toggle row when that toggle is enabled. Uses `mt-4` top margin. Wraps in a `div` with `px-5 pb-2` to give left-aligned indented feel under the toggle.

```
div.space-y-2.mt-4.px-5.pb-2
  label: text-[10px] font-black text-zinc-400 uppercase tracking-widest mb-2 ml-1
  input[type=number]: w-full px-5 py-3.5 bg-zinc-50 border border-zinc-200 rounded-xl
                      text-sm text-zinc-950 placeholder-zinc-400
                      focus:outline-none focus:ring-2 focus:ring-primary focus:border-primary
                      transition-all font-black text-lg
  p: text-[9px] font-bold text-zinc-400 uppercase tracking-widest ml-1  ← helper text
```

**Pick-up ETA field:**
- Label: `ESTIMATED PICK-UP TIME (MIN)`
- `type=number`, `min=1`, `max=120`, default value: `20`
- Helper: `MINUTES`
- Shown only when `pickup_enabled === true`

**Delivery fee field:**
- Label: `DELIVERY FEE`
- `type=number`, `min=0`, `step=0.01`, `placeholder="0.00"`
- Helper: `DISPLAYED AS CURRENCY TO CUSTOMER`
- Input displays dollar amount (e.g. `2.50`); converted to cents on save via `Math.round(value * 100)`
- Shown only when `delivery_enabled === true`

### Validation Error State

Reuse the existing top-of-form error banner pattern exactly:

```
bg-red-50 border border-red-100 rounded-[1rem] px-8 py-4 text-sm font-bold text-red-600
flex items-center justify-between shadow-sm
  div.flex.items-center.gap-3
    AlertCircle: w-5 h-5
    {error message}
  button: p-2 hover:bg-red-100 rounded-full transition-colors  ← dismiss "✕"
```

Error message copy: `"At least one order type must be active."`

Trigger condition: user attempts `handleSave` when `dine_in_enabled === false && pickup_enabled === false && delivery_enabled === false`.

## Interaction Contract

### State Machine — Save Button

| Condition | Button State | Label |
|-----------|-------------|-------|
| All three toggles off | Disabled visually; submit blocked; error shown | "Apply Settings" |
| Normal (at least one on) | Enabled | "Apply Settings" |
| `loading === true` | Disabled + spinner text | "Processing..." |
| `saved === true` (2.5s) | `bg-green-500 text-white` | `<CheckCircle2 /> Synced` |

The save button is NOT moved — it stays in the existing "System Audit" dark card at the bottom of the right column. The new "Order Types" section is added to the left column, below the "Custom Domain" section.

### Conditional Field Reveal

No animation required — toggling `pickup_enabled` or `delivery_enabled` shows/hides the conditional input via a simple JS conditional render (`{form.pickup_enabled && <div>...</div>}`). No CSS transition on the reveal. This matches the project's existing pattern for conditional UI (domain verification feedback block).

### Section Placement

Left column (`div.space-y-10` first child):
1. Regional
2. Contact Information
3. Operations (existing — item notes + KDS thresholds)
4. **Order Types** ← NEW — inserted here, below Operations
5. Custom Domain

The section always renders — no feature flag, no `<details>` collapse. Source: CONTEXT.md "No additional feature flag — the toggles themselves are the feature flags; section always visible."
