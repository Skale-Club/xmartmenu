# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Public Menu — Option Selectors + Cart.
> Generated by gsd-ui-researcher. Consumed by gsd-planner, gsd-executor, gsd-ui-auditor.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS v4 utility-first) |
| Preset | not applicable |
| Component library | none (inline Tailwind classes, no Radix/shadcn) |
| Icon library | lucide-react (^1.7.0) |
| Font | system-ui (Next.js default, no custom font declared) |

**Key constraint:** No shadcn. All interactive elements are hand-authored with Tailwind v4 classes, following the patterns established in `MenuPage.tsx`. Extend in place — do not introduce new component primitives.

## Spacing Scale

All values are multiples of 4. Source: existing Tailwind usage in `MenuPage.tsx`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap, dot indicators, inline tag padding |
| sm | 8px | Gap between radio/checkbox and label, gap between option rows |
| md | 16px | Internal padding for option group section, button horizontal padding |
| lg | 24px | Vertical gap between option groups, modal body section padding |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets for radio buttons and checkboxes: minimum 44px tall (py-2.5 applied to the wrapping label row so the full row is tappable, not just the control)
- Cart +/- buttons: 32px × 32px (`w-8 h-8`) — matches existing CartModal pattern, preserve as-is

## Typography

Source: established in `MenuPage.tsx`. Phase 6 adds no new type roles — all new text uses the existing scale.

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|-----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold` |
| Heading (modal section) | 14px | 700 (bold) | 1.4 | `text-sm font-bold` |
| Price display | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` |

**Option group name:** rendered as `text-sm font-bold text-zinc-900` — matches category headings in the menu list.

**Option name (radio/checkbox label):** rendered as `text-sm text-zinc-900` — body weight, full contrast.

**Price modifier label (e.g. "+ R$2.00"):** rendered as `text-sm text-zinc-500` inline after option name, separated by a space.

**Unit price preview line:** rendered as `text-2xl font-bold` using `accentColor` (inline style) — matches the existing product price display in `ProductModal`.

**Selected options summary in CartModal:** rendered as `text-xs text-zinc-500` on a second line below the product name — does not compete with product name weight.

## Color

Source: existing `MenuPage.tsx` palette. No new colors are introduced in this phase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fafafa` (zinc-50) | Page background (`bg-zinc-50`) |
| Secondary (30%) | `#ffffff` (white) | Modal backgrounds, cart popup, option group sections |
| Accent (10%) | `tenant.settings.accent_color` (dynamic, default `#FF5722`) | Price displays only |
| Primary brand | `tenant.settings.primary_color` (dynamic, default `#000000`) | Header background, active category pills |
| Destructive | `#ef4444` (red-500) | Remove item button in CartModal (existing `text-red-500`) |

Accent reserved for:
- Product base price display in `ProductModal` (existing, unchanged)
- Computed unit price preview line in `ProductModal` (new in this phase)
- Price display in `ProductCard` (existing, unchanged)

**Do NOT apply accentColor to:** option group headers, radio/checkbox controls, "Add to cart" button, cart total label. These use `zinc-900` (dark) as the established pattern.

**Disabled state:** when "Add to cart" is blocked by an unmet required group, use `opacity-50` on the button — same pattern as `disabled:opacity-50` already on "Confirm order" in `CartModal`.

**Selected radio/checkbox option row highlight:** `bg-zinc-50` background on the selected row (subtle, not accent) to avoid visual noise on a customer-facing screen. Border: `border-zinc-200` → selected: `border-zinc-900` on the entire row wrapper, 1px.

## Feature Gate Rules

These are not UI choices — they are hard gates that control which UI sections render.

| Gate | Condition | UI Effect |
|------|-----------|-----------|
| `directOrdersEnabled` | `settings.direct_orders_enabled === true` | Option selectors render inside `ProductModal`; "Add to cart" button renders; CartModal is accessible |
| `directOrdersEnabled === false` | — | `ProductModal` shows info only (product name, description, price, WhatsApp button). No option groups section. No "Add to cart" button. |
| `ordersEnabled` | `settings.orders_enabled === true` | Entire cart and order flow is active |
| Product has no option groups | `optionGroups.length === 0` | Option selectors section is omitted entirely from `ProductModal`; "Add to cart" fires immediately (existing behavior) |

## ProductModal — Option Selectors Section

This section is injected **between** the product description and the price/CTA row in `ProductModal`, when `directOrdersEnabled === true` AND the product has at least one option group.

### Layout

```
[Product image carousel — unchanged]
[Product name + close button — unchanged]
[Product tags — unchanged]
[Product description — unchanged]
─────────────────────────────────────
[Option selectors section — NEW]         ← inserted here
─────────────────────────────────────
[Price preview line — UPDATED]           ← replaces static price display when options exist
[CTA buttons row — UPDATED]
```

### Option Group Container

Each option group renders as a distinct block with 24px (`space-y-6`) vertical gap between groups.

Group block structure:
```
[Group name]              text-sm font-bold text-zinc-900, mb-2
[Required badge]          if required: inline "Required" badge — text-xs bg-red-50 text-red-600 px-1.5 py-0.5 rounded ml-2
[Option rows]             space-y-1 (8px gap between rows)
```

### Single-Type Group (radio)

Each option row is a full-width label wrapping a hidden native `<input type="radio">` plus visible custom control.

Row structure:
```
<label> (full row, min-height 44px, flex items-center justify-between, cursor-pointer, rounded-xl px-3 py-2.5)
  Left side: [custom radio circle] [option name] [price modifier if non-zero]
  Right side: [base_price if absolute, else nothing]
</label>
```

Custom radio circle:
- Unselected: `w-4 h-4 rounded-full border-2 border-zinc-300 flex-shrink-0`
- Selected: `w-4 h-4 rounded-full border-2 border-zinc-900 bg-zinc-900 flex-shrink-0` (filled solid zinc-900)

Row background:
- Unselected: `bg-white border border-zinc-200 rounded-xl`
- Selected: `bg-zinc-50 border border-zinc-900 rounded-xl`

Price modifier display (additive, `price_modifier !== 0`): `text-sm text-zinc-500` inline after option name — e.g., "+ R$ 2.00" or "- R$ 1.00"

Absolute price display (`base_price` is not null): right-aligned `text-sm font-semibold text-zinc-900` at end of row

### Multiple-Type Group (checkboxes)

Identical visual treatment to single-type, but uses `<input type="checkbox">` and a square custom control.

Custom checkbox:
- Unselected: `w-4 h-4 rounded border-2 border-zinc-300 flex-shrink-0`
- Selected: `w-4 h-4 rounded border-2 border-zinc-900 bg-zinc-900 flex-shrink-0 flex items-center justify-center` with a white checkmark SVG (12×12px, stroke-width 2.5)

Max reached: when `selectedCount === max_selections`, all unselected checkboxes get `opacity-40 pointer-events-none` (disabled visual, not native `disabled` to preserve layout stability).

Min/max hint: if `min_selections > 0` or `max_selections` is set, render a helper line below the group name: `text-xs text-zinc-400` — e.g., "Choose up to 3" or "Choose at least 1".

### Half-and-Half Group

Two stacked sub-selectors sharing the same option list, separated by 12px vertical gap.

Each sub-selector:
- Label above: `text-xs font-semibold text-zinc-500 uppercase tracking-wide mb-1`
  - First: "[Group name] — 1st half"
  - Second: "[Group name] — 2nd half"
- Renders identical to single-type radio group (same row styles, same custom radio control)
- The two sub-selectors are visually grouped inside a `rounded-2xl border border-zinc-100 bg-zinc-50 p-3` container

Price resolution display: when both halves are selected, show below the container: `text-xs text-zinc-500` — "Price: [formatted max price]"

Required gate: both halves must be selected before the required group is satisfied.

### Price Preview Line (replaces static price when options exist)

When the product has option groups AND `directOrdersEnabled === true`:

- Replace the static `<p style={{ color: accentColor }} className="text-2xl font-bold">` with a reactive unit price preview.
- The preview shows `formatPrice(computedUnitPrice, currency)` using `accentColor` and `text-2xl font-bold` — same visual treatment as the existing static price.
- Below the computed price, if options modified the base price, show: `text-xs text-zinc-400` — "Base: [formatPrice(product.price, currency)]"

When no option groups exist: keep the existing static price display unchanged.

## CartModal — Updated CartItem Display

### Selected Options Summary

Below the product name line in each cart item row, render a second line showing selected options:

```
[Product name]                     font-medium text-zinc-900 truncate
[Options summary]                  text-xs text-zinc-500 mt-0.5
[Unit price "each" line]           text-sm text-zinc-500
```

Options summary format: comma-separated `key: value` pairs from `Object.entries(item.selectedOptions)`. Render as a single `<p>` with `truncate` overflow handling. If `selectedOptions` is empty or all values are falsy, omit the line entirely.

Example: "Size: Large, Topping: Extra cheese"

### Total Calculation Update

The `total` line inside `CartModal` reads `item.unitPrice * item.quantity` (not `item.product.price`). The display format (`text-xl font-bold text-zinc-900`) is unchanged.

### Cart Bubble (floating button)

The existing cart bubble (`fixed bottom-20 right-4 z-40 bg-zinc-900 text-white`) is unchanged. It already shows `cartTotal` — this now reflects `unitPrice`-based totals automatically once `CartItem` is updated.

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Required group unsatisfied | On modal open or after user interaction if required group still unselected | "Add to cart" button: `opacity-50 cursor-not-allowed` (native `disabled` attribute set) |
| Required group satisfied | All required groups have valid selection | "Add to cart" button: `bg-zinc-900 text-white hover:bg-zinc-800` (active, enabled) |
| Max selections reached | `selectedCount === max_selections` in multiple group | Unchecked options: `opacity-40 pointer-events-none` |
| Loading / no options | `optionGroups` is empty array | Option selectors section omitted; "Add to cart" enabled immediately |
| Half-and-half: one half selected | First half chosen, second not yet | Second sub-selector is visible and active (no disabled state — both are always visible per D-10) |

## Copywriting Contract

All copy is multi-language via the existing `UI_COPY` record. New strings must be added to all 6 languages (en, pt, es, fr, de, it).

| Element | English Copy |
|---------|-------------|
| Option group "Required" badge | "Required" |
| Multiple group helper (max only) | "Choose up to {max}" |
| Multiple group helper (min only) | "Choose at least {min}" |
| Multiple group helper (both) | "Choose {min}–{max}" |
| Half-and-half sub-label 1st | "{groupName} — 1st half" |
| Half-and-half sub-label 2nd | "{groupName} — 2nd half" |
| Half-and-half price line | "Price: {formattedPrice}" |
| Price preview base hint | "Base: {formattedPrice}" |
| "Add to cart" button | "Add to cart" (existing — no change) |
| Options summary "each" in CartModal | "{formattedPrice} each" (existing — no change) |
| Selected options line prefix format | "{key}: {value}" (comma-separated) |

**Empty state:** Cart item with no selected options — omit the options summary line entirely. Do not render "No options selected" or any placeholder text.

**Error state (validation):** When "Add to cart" is blocked by an unmet required group, do NOT show a toast or inline error. The button disabled state is sufficient feedback. Reasoning: customers can see the unfilled required group visually.

**Destructive action (remove from cart):** The existing "✕" button in CartModal removes the item with no confirmation dialog. This is a low-stakes action (cart is in-memory, product can be re-added). No confirmation copy needed.

## Accessibility Notes

- All `<label>` elements wrap their `<input>` controls (not `for`/`id` pairing) — consistent with the existing button/input pattern in the codebase.
- Native `<input type="radio" name="{groupId}">` groups ensure keyboard navigation between options works natively.
- Native `<input type="checkbox">` elements are visually hidden (`sr-only` or `opacity-0 absolute`) but remain in the DOM so screen readers announce state.
- Disabled checkboxes when max is reached must still receive focus but announce "dimmed" or "unavailable" via `aria-disabled="true"` on the wrapper (not `disabled` on input, to avoid breaking group keyboard navigation).
- "Add to cart" button must have `aria-disabled="true"` when disabled, not just CSS `opacity-50`.
