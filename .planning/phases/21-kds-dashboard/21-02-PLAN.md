Add localStorage persistence and `toggleView` helper to `OrdersClient`. These additions go inside the `OrdersClient` default export function body, alongside the existing state declarations:

**1. Add the KDS_VIEW_KEY helper (at module level, outside the component, after the ADVANCE_LABEL constant):**
```tsx
const KDS_VIEW_KEY = (tenantId: string) => `kds_view_${tenantId}`
```

**2. Add useEffect inside `OrdersClient` to read saved preference on mount (SSR-safe):**

Insert after the `const [view, setView] = useState<'grid' | 'list'>('grid')` line:

```tsx
useEffect(() => {
  const saved = localStorage.getItem(KDS_VIEW_KEY(tenantId))
  if (saved === 'grid' || saved === 'list') setView(saved)
}, [tenantId])
```

CRITICAL — SSR-safety rule: localStorage MUST only be accessed inside `useEffect`. It MUST NOT appear in the `useState` initial value, in the render body, or behind a `typeof window !== 'undefined'` guard. The `useEffect` pattern is the correct guard for Next.js App Router.

**3. Add `toggleView` function inside `OrdersClient` (alongside `updateStatus`):**
```tsx
function toggleView(next: 'grid' | 'list') {
  setView(next)
  localStorage.setItem(KDS_VIEW_KEY(tenantId), next)
}
```
