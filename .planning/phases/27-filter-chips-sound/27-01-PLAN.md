# Phase 27 Plan 01: Filter Chips + Sound Alert

## Objective

Add KDS status filter chips and Web Audio API sound alert for new orders to OrdersClient.tsx.

## Context

- Phase 26 shipped: migration 027, useElapsedTime parameterised, StoreClient KDS section
- OrdersClient.tsx already has: Realtime subscription, polling fallback, grid/list toggle, localStorage view persistence pattern (`kds_view_{tenantId}`)
- Requirements from v1.8 Roadmap decisions:
  - Filter chips are mutually exclusive (not multi-select); default active set is [pending, preparing]
  - Web Audio API beep fires only on Realtime INSERT events (new orders), not status updates
  - AudioContext created lazily on first user interaction to satisfy browser autoplay policy
  - localStorage keys: `kds_filter_{tenantId}` and `kds_mute_{tenantId}` consistent with `kds_view_{tenantId}`

## Tasks

### Task 1: Filter chips — UI + state + localStorage persistence (KDS-10, KDS-11)

**Type:** auto

Add filter chips above the KDS grid/list. Four chips: Pendentes (pending), Em preparo (preparing), Prontos (ready), Todos. Default: ['pending', 'preparing']. Filter applied locally on `orders` state. Active filter persisted to `kds_filter_{tenantId}` localStorage key. Restored on mount via useEffect (SSR-safe, same pattern as view preference).

- Add `KDS_FILTER_KEY` constant
- Add `activeFilter` state (`'pending' | 'preparing' | 'ready' | 'all'`)
- Add `useEffect` to restore from localStorage on mount
- Derive `filteredOrders` from `orders` + `activeFilter`
- Add chip row in JSX above the grid/list
- Use filteredOrders in grid/list render instead of orders
- Count in header shows filteredOrders.length / orders.length

**Done criteria:** Four chips render; clicking each filters visible cards; Todos shows all; default hides done/cancelled; selection survives reload.

### Task 2: Sound alert + mute button (KDS-12, KDS-13)

**Type:** auto

Add Web Audio API beep on new pending Realtime INSERT. Add Bell/BellOff mute button in header. Muted state persisted to `kds_mute_{tenantId}`.

- Add `KDS_MUTE_KEY` constant
- Add `muted` state (boolean, default false)
- Add `useEffect` to restore muted from localStorage on mount
- Add `audioCtxRef = useRef<AudioContext | null>(null)` for lazy AudioContext
- Add `playBeep()` function: creates AudioContext lazily, oscillator at 880Hz, 0.1s envelope
- In Realtime INSERT handler: call `playBeep()` only if `!muted` and new order status is `pending`
- Add Bell/BellOff button in header (from lucide-react) that toggles muted and persists to localStorage

**Done criteria:** New pending order from Realtime triggers audible beep if unmuted; mute button silences it; muted state survives reload.

## Verification

- TypeScript: `npx tsc --noEmit` — 0 errors
- Filter chips render in KDS header
- Default filter hides done/cancelled orders
- localStorage keys `kds_filter_*` and `kds_mute_*` are set on interaction

## Success Criteria

1. KDS header shows four chips — Pendentes, Em preparo, Prontos, Todos — only pending+preparing visible by default
2. Active filter chip survives page reload (localStorage)
3. New pending order via Realtime INSERT produces audible beep (if unmuted)
4. Mute/unmute button silences/restores beep; state survives reload
