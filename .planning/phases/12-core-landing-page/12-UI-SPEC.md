## Spacing Scale

Declared values (multiples of 4):

| Token | px Value | Tailwind Class | Usage |
|-------|----------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, inline padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, microcopy gaps |
| md | 16px | `p-4` / `gap-4` | Default element spacing, button padding |
| lg | 24px | `p-6` / `gap-6` | Card padding, section-internal spacing |
| xl | 32px | `p-8` / `gap-8` | Layout gaps, nav horizontal padding |
| 2xl | 48px | `py-12` / `gap-12` | Section vertical padding (mobile) |
| 3xl | 64px | `py-16` / `gap-16` | Section vertical padding (desktop) |

Exceptions:
- Touch targets: minimum 44px height for all interactive elements on mobile (CTA button, nav button, FAQ accordion triggers). Use `min-h-[44px]` where needed.
- Max-width container: `max-w-5xl` (896px) for content columns, `max-w-2xl` (672px) for FAQ and narrow centered content. Apply `mx-auto px-4 sm:px-6` for responsive gutters.
- Hero vertical padding: 96px (py-24) — exceeds standard scale ceiling of 64px to achieve sufficient above-the-fold breathing room on a marketing hero section. Approved exception.

## Interaction Contracts

### CTA Buttons
- All CTA buttons are plain `<a href="/auth/register">` anchor tags styled as buttons.
- No JavaScript event handlers on any above-the-fold element.
- Hover state: `hover:bg-zinc-800` (primary dark button) or `hover:bg-zinc-100` (inverted white button).
- No loading states, no disabled states — static links only.
- Transition: `transition-colors duration-150` (CSS only, no JS animation).

### Sticky Navigation
- Implemented as `position: sticky` via Tailwind `sticky top-0` — CSS only.
- No scroll listeners, no JavaScript, no `IntersectionObserver`.
- Logo text links to `#` (same page) or `/` (harmless — page is already `/`).

### FAQ Accordion
- Implemented with native `<details>/<summary>` HTML — zero JavaScript.
- All items start collapsed (`details` without `open` attribute).
- No animation on open/close — `transition: none`. CSS height animation on `<details>` content is unreliable across browsers; the native snap-open is acceptable for this use case and preserves the no-JS constraint.
- Each `<summary>` has `list-none` to remove the default disclosure triangle; chevron icon is positioned right via flexbox.

### Social Icons in Footer
- Plain `<a href="#">` with `aria-label` attributes:
  - `aria-label="Instagram"` for Instagram icon
  - `aria-label="WhatsApp"` for WhatsApp icon
- `href="#"` placeholder until real social handles are confirmed.

### Accessibility Minimum
- All images have explicit `alt` attributes (if any images are added).
- All icon-only links have `aria-label`.
- Color contrast: zinc-900 on white = 19:1 (WCAG AAA). Zinc-600 on white = 7.4:1 (WCAG AA). Zinc-400 on zinc-900 = 4.5:1 (WCAG AA minimum).
- Focus ring: `focus:outline-none focus:ring-2 focus:ring-zinc-900 focus:ring-offset-2` on all interactive elements.
- No `tabindex` manipulation — natural DOM order.

## Anti-Pattern Enforcement

The following patterns are explicitly prohibited in Phase 12 implementation. Any plan or code that contains these must be rejected:

| Prohibited | Rationale |
|------------|-----------|
| `'use client'` in any above-fold component | Kills Lighthouse TBT — D-25, Pitfall 4 |
| Fake testimonials or fake metrics | FTC enforcement risk — CONTEXT anti-patterns, FEATURES.md |
| Crossed-out anchor pricing | Deceptive pricing, anti-pattern — REQUIREMENTS.md out-of-scope |
| Auto-play video or GIF | Mobile UX harm — CONTEXT anti-patterns |
| `next/script` for JSON-LD | RSC hydration duplication in React 19 — Pitfall 5 |
| AI seeding described as self-serve tenant tool | Dishonest — must be "our team does it" framing — D-15 |
| Ordering described as default-on | Feature-flagged per tenant — must use "available as an add-on" — D-15 |
| Supabase `getUser()` on `/` | Adds 50–200ms TTFB — middleware bypass required — D-04, Pitfall 2 |
| OG image PNG over 300 KB | WhatsApp silently drops — must be JPEG ≤ 300 KB — D-22, Pitfall 3 |
| `src/app/demo/` folder | Shadows `(public)/[slug]` — Pitfall 3 |
| Social handles with real URLs before confirmed | Use `href="#"` until real handles exist — D-20 |
