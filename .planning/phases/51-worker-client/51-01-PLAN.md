    - When enabled + Xphere returns 200 with `{account_id, contact_id, opportunity_id}` → returns `{ disabled: false, response: XphereSyncResponse }`
    - 500/502/503/429 response → throws `XphereTransientError` (status carried)

    - fetch network error / AbortSignal timeout → throws `XphereTransientError`
  </behavior>
  <action>
    Create `src/lib/xphere/client.ts` (English comments; no barrel). Import `XphereSyncRequest`, `XphereSyncResponse` from `./types` and `XphereTransientError`, `XpherePermanentError` from `./errors`.

    Define a typed result: `export type PostXphereResult = { disabled: true } | { disabled: false; response: XphereSyncResponse }`.

    Define `export function isXphereEnabled(): boolean` returning `!!(process.env.XPHERE_API_URL && process.env.XPHERE_API_KEY) && process.env.XPHERE_SYNC_ENABLED !== 'false' && !!process.env.XPHERE_SYNC_ENABLED` — i.e. both creds present AND `XPHERE_SYNC_ENABLED` is a truthy non-'false' value (kill switch). Mirror the `hasUpstash` shape in rate-limit.ts.

    Define `export async function postXphereSync(payload: XphereSyncRequest, opts?: { idempotencyKey?: string }): Promise<PostXphereResult>`:
    1. If `!isXphereEnabled()` → `return { disabled: true }` (ships dark, no fetch).
    2. Build headers: `Authorization: \`Bearer ${process.env.XPHERE_API_KEY}\``, `Content-Type: application/json`, and `X-Org-Id: process.env.XPHERE_ORG_ID` (org id sent as header). If `opts?.idempotencyKey` provided, add `Idempotency-Key` header (exact header name not finalized by Xtimator — isolated here; safe to send).
    3. `const res = await fetch(process.env.XPHERE_API_URL!, { method: 'POST', headers, body: JSON.stringify(payload), signal: AbortSignal.timeout(10_000) })` wrapped in try/catch — a thrown fetch/abort error → `throw new XphereTransientError('xphere request failed', { status: undefined, cause: err })`.
    4. On response: if `res.ok` → parse JSON as `XphereSyncResponse`, `return { disabled: false, response }`.
    5. Else classify by status: `res.status >= 500 || res.status === 429` → `throw new XphereTransientError(\`xphere ${res.status}\`, { status: res.status })`; any other non-ok (4xx) → `throw new XpherePermanentError(\`xphere ${res.status}\`, { status: res.status })`.
    Single attempt only — do NOT add a retry loop (QStash owns retries). Do NOT log the API key or full headers.
  </action>
