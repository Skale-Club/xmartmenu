<objective>
Build the keystone transport: `POST /api/internal/xphere-sync` (Node runtime). It reads the raw body once, verifies the QStash signature (current + next signing keys, pinned public URL — NOT `req.url`) BEFORE any parsing → 401 on failure; then zod-parses `{ tenantId, reason }`, fat-reads live tenant + store-admin owner profile + subscription + plan via the service-role client, resolves MRR via `getTenantPlan()`, maps with the Phase 50 `buildSyncPayload`, calls `postXphereSync` (Plan 01 seam), writes back CRM ids + `xphere_synced_at` on success (clearing `xphere_sync_error`) or a scrubbed `xphere_sync_error` on failure, and classifies the result for QStash: transient → 500, permanent → 489 + `Upstash-NonRetryable-Error: true`, disabled/gone → 2xx. Add an early `/api/internal/` passthrough in middleware so the unauthenticated QStash POST is never session-gated.

Purpose: This is the single point that turns a thin `{ tenantId, reason }` message into a verified, idempotent, retry-classified CRM upsert — mirroring the proven Stripe webhook discipline (verify-first, service-role fat-read, record success only after the business logic succeeds, else non-2xx to retry).
Output: `src/app/api/internal/xphere-sync/route.ts`, updated `src/middleware.ts`.
</objective>

    Create `src/app/api/internal/xphere-sync/route.ts`. English comments. Structure (mirror the Stripe webhook discipline):

    1. `export const runtime = 'nodejs'` — REQUIRED: the QStash `Receiver` needs Node crypto, the route needs the service-role key + fetch budget. Never Edge.

    2. Imports: `NextRequest`, `NextResponse` from `next/server`; `Receiver` from `@upstash/qstash`; `z` from `zod`; `createServiceClient` from `@/lib/supabase/server`; `getTenantPlan` from `@/lib/tenant-plan`; `buildSyncPayload` from `@/lib/xphere/mapping`; `postXphereSync` from `@/lib/xphere/client`; `XphereTransientError`, `XpherePermanentError` from `@/lib/xphere/errors`; `captureSecurityEvent` from `@/lib/observability`.

    3. zod schema: `const bodySchema = z.object({ tenantId: z.string().uuid(), reason: z.enum(['onboarded','plan_activated','plan_changed','past_due','churned','connect_changed','backfill','manual']), eventId: z.string().optional(), tags: z.array(z.string()).optional() })` (reason enum MUST match the SyncReason union in types.ts).

    4. `export async function POST(req: NextRequest)`:

    a. Read the raw body ONCE: `const rawBody = await req.text()`. Read the signature: `const signature = req.headers.get('upstash-signature') ?? ''` (lower-case; some platforms lower-case headers).

    b. Pin the public URL: `const workerUrl = process.env.XPHERE_WORKER_URL ?? \`${process.env.NEXT_PUBLIC_APP_URL}/api/internal/xphere-sync\``. Verify with both keys:
    ```ts
    const receiver = new Receiver({
      currentSigningKey: process.env.QSTASH_CURRENT_SIGNING_KEY!,
      nextSigningKey: process.env.QSTASH_NEXT_SIGNING_KEY!,
    })
    let valid = false
    try { valid = await receiver.verify({ signature, body: rawBody, url: workerUrl }) } catch { valid = false }
    if (!valid) {
      captureSecurityEvent('Xphere worker: QStash signature verification failed')
      return NextResponse.json({ error: 'invalid signature' }, { status: 401 })
    }
    ```
    Verify BEFORE any JSON.parse or DB read. Verify against `workerUrl` (the pinned constant), NEVER `req.url` (Coolify proxy rewrites host — Pitfall 10).

    c. Parse only AFTER verify: `const parsed = bodySchema.safeParse(JSON.parse(rawBody))`. On failure → this is a permanent malformed message: `return NextResponse.json({ error: 'bad payload' }, { status: 489, headers: { 'Upstash-NonRetryable-Error': 'true' } })`. Otherwise destructure `{ tenantId, reason, eventId, tags }`.

    d. Wrap the business logic in try/catch. Inside:
       - `const supabase = createServiceClient()`.
       - Fat-read the tenant: `.from('tenants').select('id, slug, name, custom_domain, xphere_sync_error').eq('id', tenantId).single()`. If the tenant row is genuinely gone → `return NextResponse.json({ skipped: 'tenant not found' }, { status: 200 })` (no retry — a deleted tenant is not a transient failure).
       - Fat-read the store-admin owner profile: `.from('profiles').select('full_name, role').eq('tenant_id', tenantId).eq('role', 'store-admin').limit(1).maybeSingle()` → `owner` (null tolerated, the mapper handles it).
       - Fat-read currency: `.from('tenant_settings').select('currency').eq('tenant_id', tenantId).maybeSingle()` → `currency = settings?.currency ?? 'brl'`.
       - Resolve the override/grandfather MRR via the plan: `const plan = await getTenantPlan(tenantId, supabase)`. If `!plan` → permanent (no subscription to sync): return `489` + `Upstash-NonRetryable-Error: true`.
       - Build the payload: `const payload = buildSyncPayload({ tenant, owner, plan, currency, reason, eventId, tags })`.
       - Call the seam: `const result = await postXphereSync(payload, { idempotencyKey: \`${tenantId}:${reason}\` })`.
       - If `result.disabled` → ships dark: `return NextResponse.json({ skipped: 'xphere disabled' }, { status: 200 })` (no retry).
       - On success, write back (FND-05): `.from('tenants').update({ xphere_account_id: result.response.account_id, xphere_contact_id: result.response.contact_id, xphere_opportunity_id: result.response.opportunity_id, xphere_synced_at: new Date().toISOString(), xphere_sync_error: null }).eq('id', tenantId)`. Then `return NextResponse.json({ ok: true }, { status: 200 })`.

    e. catch (err):
       - Scrub before persisting (Pitfall: no secrets/headers/JWT in xphere_sync_error): `const msg = err instanceof Error ? \`${err.name}: ${err.message}\`.slice(0, 500) : 'unknown error'`.
       - Persist the error and do NOT advance xphere_synced_at: `await createServiceClient().from('tenants').update({ xphere_sync_error: msg }).eq('id', tenantId)` (best-effort; ignore its own error).
       - Classify (FND-06): if `err instanceof XpherePermanentError` → `return NextResponse.json({ error: msg }, { status: 489, headers: { 'Upstash-NonRetryable-Error': 'true' } })` (DLQ). Otherwise (XphereTransientError or any unexpected throw) → `return NextResponse.json({ error: msg }, { status: 500 })` (QStash retries).

    Idempotency (FND-06): the upsert-by-`external_id = tenants.id` at the Xphere endpoint is authoritative; thin-message + fat-read makes redelivery/out-of-order self-correct (a late retry re-sends current truth). Do NOT add a local dedup ledger here — QStash deduplicationId is added by the Phase 52 producer.
