# Milestone v1.8: KDS+

**Status:** ✅ SHIPPED 2026-05-08
**Phases:** 26-27
**Total Plans:** 2

## Overview

Completion of SEED-007 Phase B — KDS operational refinements. Per-tenant configurable time thresholds (amber/red urgency colors), filter chips to hide done/cancelled orders by default, and a Web Audio API beep alerting kitchen staff when new orders arrive, with a persistent mute button.

## Phases

### Phase 26: Schema + Settings

**Goal:** Kitchen staff see order urgency colours driven by per-tenant thresholds that admins control in Store Settings.
**Requirements:** KDS-07, KDS-08, KDS-09
**Plans:** 1 plan

Plans:
- [x] 26-01: Migration 027 (`amber_threshold_minutes` + `red_threshold_minutes` on `tenant_settings`, defaults 10/20) + TypeScript types + `useElapsedTime` hook parameterized + `OrdersClient` prop threading + Store Settings KDS section

**Key decisions:**
- `useElapsedTime(createdAt, amberMinutes=10, redMinutes=20)` — default params maintain backward compatibility
- `orders/page.tsx` fetches `tenant_settings` in `Promise.all` alongside orders; fallback `?? 10` / `?? 20` if settings row missing
- StoreClient validation fires before upsert: `amber >= red` or either `<= 0` blocks save with inline error
- Both inputs `type="number" min={1} max={120}` — client-side HTML constraint + server-side JS guard

### Phase 27: Filter Chips + Sound

**Goal:** Kitchen staff can focus on the orders that need attention and are alerted the instant a new order arrives.
**Requirements:** KDS-10, KDS-11, KDS-12, KDS-13
**Plans:** 1 plan

Plans:
- [x] 27-01: Filter chips (Ativos/Pendentes/Em preparo/Prontos/Todos, default Ativos=pending+preparing) + `kds_filter_{tenantId}` localStorage + `playBeep()` Web Audio API on INSERT + Bell/BellOff mute button + `kds_mute_{tenantId}` localStorage

**Key decisions:**
- Added `'active'` filter value (pending+preparing combined) as default — KDS-10 spec required `['pending','preparing']` default
- `mutedRef` mirrors `muted` state via `useEffect` to avoid stale closure in Realtime handler without re-subscribing
- `playBeep()` wrapped in try/catch — `AudioContext` blocked by browser autoplay policy silently fails
- Beep only fires on INSERT events when `status === 'pending'` — no sound for UPDATE/status transitions
