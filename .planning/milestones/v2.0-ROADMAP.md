# v2.0 Monetization Roadmap

**Seed:** SEED-009 — Plans, Pricing & Stripe Connect Monetization
**Status:** In Progress
**Started:** 2026-05-09

## Overview

v2.0 delivers the monetization layer for xmartmenu:
- Subscription plans (menu/orders/payments tiers)
- Stripe Connect for direct tenant payments
- Feature gating based on plan type

## Phase List

### Phase 30: Schema + Planos Base

**Goal:** Database schema for plans, subscriptions, Stripe connections + seed data

**Success Criteria:**
1. Migration creates tables: `plans`, `tenant_subscriptions`, `stripe_connections`, `processed_stripe_events`
2. Seed 3 plans: `menu` ($49), `orders` ($99), `payments` ($179)
3. All existing tenants migrated to `payments` plan (grandfathered)
4. Types added to `src/types/database.ts`
5. `getTenantPlan(tenantId)` helper returns effective plan with overrides

**Requirements:** MON-01, MON-02

---

### Phase 31: Superadmin Plan Management

**Goal:** Superadmin UI for plan CRUD and tenant subscription overrides

**Success Criteria:**
1. `/superadmin/plans` — list all plans with inline edit (name, prices, features, is_active)
2. `/superadmin/tenants/[id]` — Subscription tab with current plan, billing cycle, override fields
3. Override fields: monthly_price, annual_price, transaction_fee_pct, notes
4. Save override button persists to tenant_subscriptions

**Requirements:** MON-01, MON-02

---

### Phase 32: Stripe Connect OAuth

**Goal:** Stripe Connect integration for payments tier tenants

**Success Criteria:**
1. `/api/stripe/connect/oauth` — initiates OAuth redirect to Stripe
2. `/api/stripe/connect/callback` — handles code exchange, saves stripe_account_id
3. `/api/stripe/connect/disconnect` — deactivates connection
4. Tenant Settings → Subscription: Connect Stripe button (payments plan only)
5. Connection status display, disconnect button

**Requirements:** MON-03, MON-04

---

### Phase 33: Payment Intent + Webhook

**Goal:** Process payments through Stripe Connect

**Success Criteria:**
1. Order creation triggers PaymentIntent for payments-plan tenants
2. `application_fee_amount` = amount × transaction_fee_pct
3. `transfer_data.destination` = tenant's stripe_account_id
4. `/api/stripe/webhook` handles: payment_intent.succeeded, payment_intent.payment_failed, charge.refunded
5. `orders.payment_status` and `orders.stripe_payment_intent_id` columns
6. Idempotency via processed_stripe_events

**Requirements:** MON-03, MON-05

---

### Phase 34: Tenant Subscription UI

**Goal:** Tenant-facing subscription management

**Success Criteria:**
1. Tenant Settings → Subscription tab: current plan, effective price, billing cycle, renewal date
2. Upgrade button shows plan comparison (prices from DB, not hardcoded)
3. Payment status badge on KDS order cards
4. Payment status visible in admin orders modal

**Requirements:** MON-01, MON-02, MON-04

---
