# Milestone v1.7: Customization

**Status:** ✅ SHIPPED 2026-05-08
**Phases:** 23-25
**Total Plans:** 5

## Overview

Full ingredient customization system for xmartmenu. Restaurant tenants can build a normalized ingredient catalog, assign ingredients to products with per-product price overrides, and let customers customize their orders McDonald's-style (remove onions, double cheese, add bacon). Modifications flow through the cart → order → DB → KDS card → admin orders modal as structured JSONB, rendered with color-coded prefixes.

Note: SEED-008 Phase D (per-item free-text notes + `item_notes_enabled`) was delivered in v1.6. v1.7 covers Phases A-C (catalog, customer panel, kitchen display).

## Phases

### Phase 23: Ingredient Schema

**Goal:** DB has normalized ingredient catalog, product composition, and order JSONB column — all with RLS and TypeScript types.
**Requirements:** INGR-01, INGR-02, INGR-03, INGR-04
**Plans:** 1 plan

Plans:
- [x] 23-01: Migration 026 — `ingredients` + `product_ingredients` + `ingredient_customization_enabled` flag + `ingredient_modifications JSONB` + TypeScript types

**Key decisions:**
- RLS uses `auth_tenant_id() OR is_superadmin()` (project convention, not `auth.jwt()`)
- Both tables get `_public_read` policy — required for public menu to display ingredients in Phase 25
- `PRIMARY KEY (product_id, ingredient_id)` on `product_ingredients` implicitly enforces uniqueness (no separate UNIQUE constraint needed)

### Phase 24: Admin Catalog

**Goal:** Store admin manages ingredient catalog and assigns ingredients to products with price overrides.
**Requirements:** INGR-05, INGR-06
**Plans:** 2 plans

Plans:
- [x] 24-01: `/admin/menu/ingredients` CRUD page + sidebar nav item gated by flag + layout.tsx flag query
- [x] 24-02: Tab bar (Detalhes/Opções/Ingredientes) in product editor + multi-select picker + is_default toggle + price override inputs

**Key decisions:**
- No @dnd-kit — ChevronUp/Down arrows for reorder (existing project pattern from ProductDetailClient)
- AdminSidebar receives `ingredientCustomizationEnabled?: boolean` prop — layout.tsx queries flag in both superadmin-preview and regular tenant paths
- Price override inputs: `defaultValue` + `onBlur` (not controlled) — empty field on blur sends `null` to DB (catalog default)
- Tab bar uses `useState<'details' | 'options' | 'ingredients'>` — "Ingredientes" tab hidden when flag is false

### Phase 25: Customer + Kitchen

**Goal:** Customers customize ingredient composition with live price update; modifications persist and render in KDS + admin orders.
**Requirements:** INGR-07, INGR-08, INGR-09, INGR-10
**Plans:** 2 plans

Plans:
- [x] 25-01: Server pages fetch `productIngredientsByProductId` + MenuPage customization panel (stepper chips, inline "Adicionar ingrediente" picker, live price delta, `buildIngredientModifications`) + cart→API→DB pipeline
- [x] 25-02: KDS OrderCard + admin orders modal render `ingredient_modifications` with color-coded text

**Key decisions:**
- `ProductModal` is inline in `MenuPage.tsx` (~1200 lines) — no separate file
- Both `[slug]/page.tsx` AND `[slug]/[menuSlug]/page.tsx` need the ingredient fetch
- `buildCartKey` unchanged — same product+options merges to same slot; modifications replace
- `buildIngredientModifications()` returns `null` when all arrays empty (not `{}`)
- "Adicionar ingrediente" is inline expandable (not a modal) — avoids z-index conflicts inside overlay
- Literal Tailwind classes: `text-red-600 line-through`, `text-amber-600`, `text-green-600`

## DB Migration 026 (applied 2026-05-08)

```sql
CREATE TABLE IF NOT EXISTS ingredients (...);  -- 11 cols, RLS
CREATE TABLE IF NOT EXISTS product_ingredients (...);  -- 7 cols, UNIQUE(product_id,ingredient_id), RLS
ALTER TABLE tenant_settings ADD COLUMN IF NOT EXISTS ingredient_customization_enabled BOOLEAN NOT NULL DEFAULT false;
ALTER TABLE order_items ADD COLUMN IF NOT EXISTS ingredient_modifications JSONB;
```

Applied via Node.js `pg` client.

## Milestone Summary

**Issues Resolved:**
- Restaurants with burgers/pizzas/açaí had no way to let customers remove/add ingredients
- Option groups (v1.1) handle "pick one of N" but not "tweak defaults" — a fundamentally different primitive
- Per-item notes (v1.6) handle free text but not structured ingredient-level pricing

**Deferred (Out of Scope):**
- Charging for ingredient removal (v1.8 — rare, complex display)
- Nutrition info / allergen tagging (future)
- WhatsApp export including ingredient modifications (future)
- Responsive image variants / AVIF (not in scope)
