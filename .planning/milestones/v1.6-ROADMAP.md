# Milestone v1.6: Operations

**Status:** ✅ SHIPPED 2026-05-08
**Phases:** 21-22
**Total Plans:** 4

## Overview

Transformed the admin orders page from a static table into a real-time Kitchen Display System (KDS). Restaurant operators now see orders as a color-coded card grid on a tablet in the kitchen, with automatic updates when new orders arrive. Customers can attach per-item notes ("sem cebola", "ponto mal passado") that flow directly to the kitchen card and admin orders view.

## Phases

### Phase 21: KDS Dashboard

**Goal:** Kitchen staff can monitor all open orders at a glance via a color-coded, timed card grid and advance order status without leaving the page.
**Requirements:** KDS-01, KDS-02, KDS-03, KDS-04, KDS-05
**Plans:** 2 plans

Plans:
- [x] 21-01: `useElapsedTime` hook + corrected `STATUS_COLORS` + `OrderCard` component + grid layout + `tenantId` prop
- [x] 21-02: localStorage grid/list toggle + optimistic status PATCH wiring

**Key decisions:**
- `STATUS_COLORS` had pending/preparing inverted in original code — corrected: `pending=blue, preparing=yellow`
- `tenantId` added to `OrdersClient` props from `page.tsx` — required for `kds_view_{tenantId}` localStorage key
- `loadingId: string | null` replaces `loading: boolean` — only in-flight card button is disabled
- Tailwind CSS 4: all status classes as full literal strings (no interpolation)

### Phase 22: Realtime + Per-Item Notes

**Goal:** New orders appear automatically on the KDS without manual refresh, and customers can attach per-item notes that flow through to the kitchen card and admin orders table.
**Requirements:** KDS-06, NOTE-01, NOTE-02, NOTE-03, NOTE-04
**Plans:** 2 plans

Plans:
- [x] 22-01: Migration 025 + TypeScript type + Realtime subscription + 15s polling + notes display
- [x] 22-02: StoreClient `item_notes_enabled` toggle + customer textarea + `sanitizeNote` API validation

**Key decisions:**
- Realtime + polling simultaneously (not fallback-only) — Realtime for instant appearance, polling replaces full state
- Realtime INSERT event carries only `orders` row — follow-up `.select('*, order_items(*)')` required before prepending to state
- `item_notes_enabled` lives on `tenant_settings` (same pattern as `direct_orders_enabled`)
- `sanitizeNote`: strip control chars `\x00-\x08\x0B-\x1F\x7F`, trim, cap 140 chars server-side
- `CartItem.note` excluded from `buildCartKey` — same product+options merges into one slot; re-add replaces the note

## DB Migration 025 (applied 2026-05-08)

```sql
ALTER TABLE tenant_settings ADD COLUMN IF NOT EXISTS item_notes_enabled BOOLEAN NOT NULL DEFAULT false;
ALTER TABLE order_items ADD COLUMN IF NOT EXISTS notes TEXT;
ALTER PUBLICATION supabase_realtime ADD TABLE orders;
```

Applied via Node.js `pg` client (same pattern as migration 024).
