# Plano: Corrigir Redirect de Superadmin para Onboarding

## Problema
Superadmins estão sendo redirecionados para onboarding incorretamente. O problema é que 3 locais não usam `normalizeRole()` para verificar o role do superadmin, então quando o role no banco é `'super-admin'` (com hífen), a comparação `'superadmin' === 'super-admin'` falha.

## Solução
Adicionar normalização de role em todos os locais que verificam superadmin.

---

## Tarefas de Correção

### 1. middleware.ts - Linha 60

**Arquivo:** `src/lib/supabase/middleware.ts`

**Mudança:** Usar `normalizeRole()` na verificação de role

```typescript
// Antes (linha 60):
const role = profile?.role

// Depois:
const role = normalizeRole(profile?.role)
```

**Também necessário:** Importar a função no topo do arquivo.

---

### 2. onboarding/page.tsx - Linha 72

**Arquivo:** `src/app/onboarding/page.tsx`

**Mudança:** Usar `normalizeRole()` na verificação client-side

```typescript
// Antes (linha 72):
if (profile?.role === 'superadmin') {

// Depois:
if (normalizeRole(profile?.role) === 'superadmin') {
```

**Também necessário:** Importar `normalizeRole` do `@/lib/auth/role-utils`.

---

### 3. api/onboarding/route.ts - Linha 76

**Arquivo:** `src/app/api/onboarding/route.ts`

**Mudança:** Usar `normalizeRole()` na verificação

```typescript
// Antes (linha 76):
if (currentProfile?.role === 'superadmin') {

// Depois:
if (normalizeRole(currentProfile?.role) === 'superadmin') {
```

**Também necessário:** Importar `normalizeRole` do `@/lib/auth/role-utils`.

---
